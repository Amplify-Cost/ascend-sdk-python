## Security Layers

```
    REQUEST FLOW
         │
         ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYER 1: RATE LIMITING                                          │
    │ • Per-user, per-org, per-endpoint limits                        │
    │ • Redis-backed with fail-secure to DENY                         │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYER 2: PROMPT SECURITY                                        │
    │ • Injection detection, jailbreak prevention                     │
    │ • Pattern-based + ML analysis                                   │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYER 3: CODE ANALYSIS                                          │
    │ • SAST scanning, secrets detection                              │
    │ • Dangerous pattern identification                              │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYER 4: ACTION GOVERNANCE                                      │
    │ • CVSS v3.1 risk scoring                                        │
    │ • Policy evaluation (ALLOW/DENY/REQUIRE_APPROVAL)               │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYER 5: JWT AUTHENTICATION                                     │
    │ • RS256 signature validation                                    │
    │ • Token expiration, claims verification                         │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYER 6: API KEY VALIDATION                                     │
    │ • Constant-time comparison                                      │
    │ • Scope and permission verification                             │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYER 7: RBAC AUTHORIZATION                                     │
    │ • 6-level role hierarchy                                        │
    │ • Organization-scoped permissions                               │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
    ┌─────────────────────────────────────────────────────────────────┐
    │ LAYERS 8-12: ENCRYPTION, AUDIT, VALIDATION                      │
    │ • BYOK encryption (AES-256-GCM)                                 │
    │ • Immutable audit logging with hash-chain                       │
    │ • Input validation, secrets management, security headers        │
    └────────────────────────────┬────────────────────────────────────┘
                                 │ PASS
                                 ▼
                           ┌──────────┐
                           │ BUSINESS │
                           │  LOGIC   │
                           └──────────┘
```

## Integration Points

| Integration | Protocol | Security |
|-------------|----------|----------|
| Web UI | HTTPS | JWT + CSRF |
| Python SDK | HTTPS | API Key |
| Kong Plugin | HTTP (internal) | mTLS |
| Envoy ext_authz | gRPC | mTLS |
| Lambda Authorizer | AWS IAM | IAM Role |
| Webhooks | HTTPS | HMAC Signature |
